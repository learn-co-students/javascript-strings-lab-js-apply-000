<h2 id="overview">Overview</h2>

<p>In this lab, we&#39;re going to work with strings. Strings in JavaScript are wrapped in single or double quotes, or in back ticks.</p>

<h2 id="objectives">Objectives</h2>

<ol>
<li>Manipulate strings in JavaScript</li>
<li>Practice interpolating with template literals</li>
</ol>

<h2 id="introduction">Introduction</h2>

<p>Imagine we&#39;re planning a birthday party for Bill Nye. There are going to be a lot of people there, so we&#39;re going to use JavaScript to help us keep everything straight.</p>

<p>First, we need to practice greeting everyone. (I don&#39;t know about you, but I sometimes get nervous and say the dumbest things — but we don&#39;t want to embarrass ourselves in front of Bill Nye!)</p>

<p>One might think that we could just type</p>
<pre class="highlight javascript"><code><span class="nx">Hello</span><span class="p">,</span> <span class="nx">everybody</span><span class="o">!</span>
</code></pre>
<p>in our browser&#39;s console and be done with it. Give it a try. (If you&#39;re on a Mac, that would be <code>Command</code> + <code>Option</code> + <code>J</code> together.)</p>

<p>You should see something like</p>
<pre class="highlight plaintext"><code>Uncaught ReferenceError: Hello is not defined(…)
</code></pre>
<p>Well, that won&#39;t work. (This is why we practice!) In order to greet our guests, we need to tell JavaScript that we&#39;re using a <strong>string</strong>. A string is a collection of characters (letters, numbers, and symbols) wrapped in single or double quotes (or, as we&#39;ll see, in back ticks). So to greet everyone, we can write,</p>
<pre class="highlight javascript"><code><span class="s1">'Hello, everybody!'</span>
</code></pre>
<p>or</p>
<pre class="highlight javascript"><code><span class="s2">"Hello, everybody!"</span>
</code></pre>
<p>Single or double quotation marks can contain a string variable..</p>

<p>What if we want to say hi to a special guest, like Neil deGrasse Tyson? When we wrap strings in single or double quotes, we can join them together using the <code>+</code> operator:</p>
<pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">specialGuest</span> <span class="o">=</span> <span class="s2">"Neil deGrasse Tyson"</span>
<span class="s2">"Hello, "</span> <span class="o">+</span> <span class="nx">specialGuest</span> <span class="o">+</span> <span class="s2">"!"</span> <span class="c1">// "Hello, Neil deGrasse Tyson!"</span>
</code></pre>
<p>This is called <em>concatenation</em>. Notice that the value of the <code>specialGuest</code> variable is <em>also</em> a string!</p>

<p><strong>TOP TIP</strong>: Your console might be getting a little full at this point. If at any point you&#39;d like to clear it out and start fresh, you can either click the button in the top left corner of the console — in Chrome, it looks like this:</p>

<p><img src="https://curriculum-content.s3.amazonaws.com/skills-based-js/clear_console.png" alt="clear console"></p>

<p>Alternatively, you can press <code>ctrl + L</code> or <code>command + K</code>. As long as you don&#39;t refresh the page, anything you&#39;ve declared will stick around for you to reference — you&#39;ll just get a nice blank slate on which to code.</p>

<p>When we wrap strings in back ticks, we can use placeholders (<code>${}</code>) and insert variables or evaluated JavaScript directly:</p>
<pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">specialGuest</span> <span class="o">=</span> <span class="s2">"Neil deGrasse Tyson"</span><span class="p">;</span>

<span class="s2">`Hello, </span><span class="p">${</span><span class="nx">specialGuest</span><span class="p">}</span><span class="s2">! High </span><span class="p">${</span><span class="mi">3</span> <span class="o">+</span> <span class="mi">2</span><span class="p">}</span><span class="s2">!`</span> <span class="c1">// "Hello, Neil deGrasse Tyson! High 5!"</span>
</code></pre>
<p>This is called <em>interpolation</em>.</p>

<h2 id="lab">Lab</h2>

<p>You&#39;ll find a file called <code>index.js</code> in this directory. Your mission, should you choose to accept it, is to get its tests (in <code>tests/index-test.js</code>) to pass.</p>

<p>You can run the tests using the <code>learn</code> command in your terminal or the Learn IDE. Give that a go now.</p>

<p><img src="https://i.giphy.com/9c830567WqLCw.gif" alt="waiting"></p>

<p>All three tests have failed! This is okay, and it&#39;s expected — you haven&#39;t written any code yet, after all.</p>

<p>In <code>index.js</code>, you&#39;ll see five lines of code:</p>
<pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">greeting</span> <span class="o">=</span> <span class="s2">""</span><span class="p">;</span>

<span class="kd">var</span> <span class="nx">specialGuest</span> <span class="o">=</span> <span class="s2">"Neil deGrasse Tyson"</span>

<span class="kd">var</span> <span class="nx">greetSpecialGuest</span> <span class="o">=</span> <span class="s2">""</span> <span class="o">+</span> <span class="nx">specialGuest</span> <span class="o">+</span> <span class="s2">"!"</span><span class="p">;</span>

<span class="kd">var</span> <span class="nx">topic</span> <span class="o">=</span> <span class="s2">"space"</span><span class="p">;</span>

<span class="kd">var</span> <span class="nx">conversation</span> <span class="o">=</span> <span class="s2">`</span><span class="p">${</span><span class="nx">topic</span><span class="p">}</span><span class="s2">`</span><span class="p">;</span>
</code></pre>
<p>Each line has a test associated with it. When the tests fail, they show us what the <em>expected</em> value is — your job is to make that expectation a reality by modifying the code provided.</p>

<p>When you first run <code>learn</code>, you should see something like this:</p>

<p><img src="https://user-images.githubusercontent.com/17556281/27979675-b6575498-6345-11e7-8c9d-052c2d4d3e96.png" alt="All tests failing."></p>

<p>Let&#39;s walk through that first error together. First, we see the test title:</p>
<pre class="highlight shell"><code>1<span class="o">)</span> strings defines <span class="sb">`</span>greeting<span class="sb">`</span>:
</code></pre>
<p>The title tells us what the test expects our code to do. In this case,
&quot;strings&quot; refers to the general problem space in which we&#39;re working —
we&#39;re handling strings.</p>

<p>Continuing on with the test output, we can now make better sense of the next few lines:</p>
<pre class="highlight shell"><code>AssertionError: <span class="s1">'!'</span> <span class="o">==</span> <span class="s1">'Hello, everybody!'</span>
+ expected - actual

- !
+Hello, everybody!
</code></pre>
<p>This is a lot to take in, so we&#39;ll go through it slowly.</p>

<p>What could <code>AssertionError</code> mean? Well, it probably means that our test <em>asserted</em> (or expected) that something would be true, and that thing wasn&#39;t true.</p>

<p>What is that thing? The test expected the empty string, <code>&#39;&#39;</code>, to be equal to the string <code>&#39;Hello, everybody!&#39;</code> — but, of course, these strings are not equal.</p>

<p><code>+ expected - actual</code> is a key for reading the statements below it. <code>+ expected</code> tells us that the expected output shows up in that yellowish green; <code>- actual</code> tells us what actually happened.</p>

<p>But reading on, we only see <code>+Hello, everybody!</code> — what&#39;s going on? Why isn&#39;t there any <code>- actual</code> output? Well, there <em>was</em> no actual output — it&#39;s just an empty string! That must be the problem!</p>

<p>Next, the title tells us that <code>index.js</code> &quot;defines <code>greeting</code>.&quot; Let&#39;s look in <code>index.js</code> — sure enough, we see, at the top of the file, <code>var greeting = &quot;&quot;;</code>. Seems like a reasonable place to start.</p>

<p>What if, instead of assigning <code>&quot;&quot;</code> to <code>greeting</code>, we assign <code>&quot;Hello, everybody!&quot;</code>, like the test expects. Go ahead and change that line in <code>index.js</code> so it reads</p>
<pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">greeting</span> <span class="o">=</span> <span class="s2">"Hello, everybody!"</span><span class="p">;</span>
</code></pre>
<p>save the file, and rerun your tests. You should see</p>

<p><img src="https://user-images.githubusercontent.com/17556281/27979674-b65255f6-6345-11e7-8fca-d71760c514ef.png" alt="First test passes!"></p>

<p>Nice! You got the first test to pass.</p>

<p>Now use the skills that you learned above to read through the rest of the test output and fix those errors, too! Always remember to save your file before re-running your tests.</p>

We&#39;ve given you some starter code to point you in the right direction — try to solve the lab as intended! You can then compare your solution with ours (found in the <code>solution</code> branch of this repository).</p>

<p>When your tests are passing, submit your answer by typing in <code>learn submit</code> in the command line or else create a pull request (use <code>learn submit</code> if &quot;pull request&quot; sounds a bit terrifying).</p>

<p>Good luck!</p>
